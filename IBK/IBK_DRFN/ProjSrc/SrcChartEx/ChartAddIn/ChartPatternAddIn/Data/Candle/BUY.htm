BUY It clears all selling over position and an order function which takes new buying over position.

BUY (
  strName,
  nType,
  nPrice,
  nContracts
);

Parameter 1 strName
[strName] Enumerate the name of signal indication.

Parameter 2 nType
[nType (optional)] Select one among below signal indication types.
nType Description
ONCLOSE Order by close price of base BAR,Order by open price of next BAR in real trading(Default value)
ATMARKET Order by open price of next BAR
ATLIMIT In case of buying over position order(BUY , EXITSHORT),Order below a limit(nPrice) in order not to be buying over entry(BUY) above specific price or selling over clearing entry(EXITSHORT).
In case of selling over position order(SELL , EXITLONG),Order over a limit(nPrice) in order not to be selling over entry(SELL) below specific price or buying over clearing entry(EXITLONG).
ATSTOP In case of buying over position order(BUY , EXITSHORT),When becomes over designated price ,puts in order.
In case of selling over position order(SELL, EXITLONG),when becomes below designated price ,puts in order.

Parameter 3 nPrice
[nPrice (optional)] Enumerate the order price.
In case of ATLIMIT or ATSTOP,an order price becomes a target price and will be ignored the order price of ONCLOSE or ATMARKET. This order price is not releation with actual order price. In ONCLOSE, ATMARKET, nPrice is disregarded(DEF transact). In ATLIMIT, ATSTOP,becomes limit price. Only the limit price which confirms the point of view of an order,the order by this order price doesn't put. Actual order price puts an order by the price which it sets from the attribute window of system trading.

Parameter 4 nContracts
[nContracts (optional)] Enumerate an order quantity. The default is DEF and an order will be put by a quantity setted in the trading. If an order quantity will be set,the order quantity of position entry order and corresponding position clearing is same.

Examples
VAR : bBuyCond(FALSE);
If bBuyCond Then Begin
  BUY("B_AB");
  EXITLONG("EL_AB");
End;
// To follow in codintion of bBuyCond, takes an order of new buying over position by ONCLOSE type and represents signal name with "B_AB"
// To follow in codintion of bBuyCond, takes an order of buylling over position clearing by ONCLOSE type and represents signal name with "EL_AB"
